# Ajax

# 1.AJAX 基础

AJAX 的应用场景，页面上拉加载更多数据，列表数据无刷新分页，表单项离开焦点数据验证，搜索框提示文字下拉列表

# 2.Ajax 运行原理

AJAX 相当于浏览器发送请求与接收响应的代理人，以实现在不影响用户浏览页面的情况下，局部更新页面数据，从而提高用户体验

# 3.Ajax 的实现步骤

# 3.1. 创建 Ajax 对象

var xhr=XMLHttpRequest();

# 3.2. 告诉 Ajax 请求地址以及请求方式

xhr.open ('get',' 网址 ')

# 3.3. 发送请求

xhr.send();

# 3.4. 获取服务器端与客户端的响应数据

xhr.onload=function (){ console.log (xhr.responseText);} 因为获取响应数据需要时间，且这个时间不确定，所以我们要添加一个 onload 函数

# 4. 服务器端响应的数据格式

在真实项目中，服务器端大多数情况下会以 JSON 对象作为响应数据的格式，当客户端拿到响应数据时，要将 JSON 数据和 HTML 字符串进行拼接，然后将拼接的结果展示在页面中

在 http 请求与响应的过程中，无论是请求参数还是响应内容，如果是对象类型，最终都会被转换为对象字符串进行转换，可以用 JSON.parse (), 将 json 字符串转换成 json 对象

# 5.get 请求参数传递

要将数据传递给服务器，我们需要自己拼接请求参数，并在配置 Ajax 对象时把要传递的参数加在 get? 后面然后在服务器里创建这个路由

# 6.post 请求参数传递

post 请求方式，post 请求参数写在 send 里面，然后 post 的请求报文用 setRequestHeader (' 请求参数的格式类型 ',)

# 7. 请求参数的格式

1.application/x-www-form-urlencoded，这种请求的参数格式为属性 1 = 值 & 属性 2 = 值，这种字符串的形式

application/json 在，这种请求的参数格式为对象 {属性名:' 属性值 '，属性名:' 属性值 '}，要用这种格式，就需要把 Content-Type 属性的值改为 application/json，告诉服务器端当前请求参数的格式是 json，但 send 里面传递的参数必须是字符串，所以我们需要把 json 对象转换成字符串，需要利用到 JSON.stringify ()，且如果要用 json 格式，则必须要用 post，get 不行

# 8. 获取服务端的响应

Ajax 状态码，在创建 ajax 对象，配置 ajax 对象，发送请求，以及接收完服务器端响应数据，这个过程中的每一个步骤都会对应一个数值，这个数值就是 ajax 状态码，有以下几个数值

0: 请求未初始化 (还没有调用 open ())

1: 表示请求已经建立，但是还没有发送 (还没有调用 send ())

2. 表示请求已经发送

3. 请求正在处理中，通常响应中已经有部分数据可以用了

4. 响应已经完成，可以获取并使用服务器的响应了

可以用 readyState 来获取状态码，onreadystatechange 事件，当 ajax 状态码发生变化时将自动触发该事件

我们可以用一个 if 来判断 Ajax 的状态码是否为 4，如果为 4 的话我们就可以调用响应报文了

# 9.Ajax 错误处理

1. 网络畅通，服务器端能接收到请求，服务器端返回的不是预期结果。遇到这种错误可以判断服务器的状态码，分别进行处理。xhr.status 获取 http 状态码

2. 网络通畅，服务器端没有接收到请求，返回 404 状态码。这种情况下就要考虑请求地址是否拼写错误

3. 网络通畅，服务器端能接收到请求，服务器端返回 500 状态码。服务器端出错，找后端人员沟通

4. 网络中断，请求无法发送到服务器端。这种情况不会触发 xhr.onload 事件，会触发 xhr.onerror 事件。在 onerror 事件处理函数中对错误进行处理

Ajax 状态码：表示 Ajax 请求的过程状态，是由 Ajax 对象返回的。HTTP 状态码：表示请求的处理过程，是由服务器端返回的

# 10. 低版本 IE 浏览器的缓存问题

在低版本的 IE 浏览器中，AJAX 有严重的缓存问题，即在请求地址不发生变化的情况下，只有第一次请求会真正发生到服务器，后续的请求都会从浏览器的缓存中获取结果，即使服务器更新了数据，客户端依旧会拿到旧的数据

解决方案：在请求地址的后面加参数且每次加的都不一样，在‘请求地址后面以？参数名’=+Math.random ()，这样就保证了每次的请求地址都不一样，但要注意的是，参数名不能和我们真正需要请求的地址里面的参数相同

# 11.Ajax 异步编程

Ajax 是异步执行的

# 11.1.Ajax 封装

封装一个 Ajax 函数，传递一个对象参数，对象里面有 type url success 三个属性，type 表示请求方式，url 表示请求地址，success 是一个函数，表示请求成功后对数据进行的处理

# 11.1.1. 拓展

请求参数：如果我们需要在请求地址后面加上请求参数，这时我们不能直接加在请求地址后面，有几个问题需要我们考虑，一是请求参数位置问题，get 是放在请求地址后面，post 是放在 send 方法中，二是请求参数格式问题，一种是 application/x-www-form-urlencoded，也就是参数名称 = 参数值 & 参数名称 = 参数值 的形式，一种是 application/json，也就是 {属性名：值，属性名：值} 的形式，我们选择了第二种，因为传递对象数据类型对于函数的调用者更加友好，而且在函数内部对象数据类型转换为字符串数据类型更方便

把 json 类型的参数转换成字符串格式，就是遍历对象，然后判断请求方式，如果是 get，就把字符串加在 url 后面，如果是 post，就需要设置请求头，并把字符串 send

上面提到的设置请求头则又涉及到格式问题，这时我们可以再在 ajax 函数的对象里添加一个 header 对象，然后判断是 x-www-form-urlencoded 还是 json，如果是 json 的话，就需要把 options.data 用 JSON.stringfy () 转换一下，如果是另一个则直接 send 就好

客户端返回的是 JSON 字符串，所以我们需要把 JSON 字符串转换成 JSON 对象，首先我们要判断是否返回的是 JSON 格式，用 xhr 下的一个函数 getResponseHeader ('Content-Type') 就可以得到返回的响应头里的 Content-Type 了，然后再判断 application/json 这个字符串是否在响应头中，用了 includes () 函数，里面的参数跟要查询的字符串

这时又有一个问题，就是我们要传递的参数太多了，此时我们可以采用以下方法，就是在 Ajax 里面设置一组默认的值，然后我们传递进去的参数就把默认的值给覆盖，没有传递的值就使用默认值，这里的覆盖用到了 Object 里的一个函数 assign ()，后面跟两个参数，第一个是要被覆盖的，第二个是要拿来覆盖的

# 12. 模板引擎

作用，使用模板引擎提供的语法，可以将数据和 HTML 拼接起来，因为服务器返回的结果大多是 json 格式的

使用步骤：1、下载 art-template 模板引擎库文件并在 HTML 页面中引入库文件，利用的是 script 引入；2、准备 art-template 模板；3、告诉模板引擎将哪一个模板和哪个数据进行拼接，当我们引入了模板引擎后，我们就可以使用 template 方法
